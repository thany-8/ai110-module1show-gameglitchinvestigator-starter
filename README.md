# 🎮 Game Glitch Investigator: The Impossible Guesser

## 🚨 The Situation

You asked an AI to build a simple "Number Guessing Game" using Streamlit.
It wrote the code, ran away, and now the game is unplayable. 

- You can't win.
- The hints lie to you.
- The secret number seems to have commitment issues.

## 🛠️ Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Run the broken app: `python -m streamlit run app.py`

## 🕵️‍♂️ Your Mission

1. **Play the game.** Open the "Developer Debug Info" tab in the app to see the secret number. Try to win.
2. **Find the State Bug.** Why does the secret number change every time you click "Submit"? Ask ChatGPT: *"How do I keep a variable from resetting in Streamlit when I click a button?"*
3. **Fix the Logic.** The hints ("Higher/Lower") are wrong. Fix them.
4. **Refactor & Test.** - Move the logic into `logic_utils.py`.
   - Run `pytest` in your terminal.
   - Keep fixing until all tests pass!

## 📝 Document Your Experience

- Purpose of the Game

The purpose of the game is to guess a secret number generated by the program. The player receives hints indicating whether the next guess should be higher or lower until the correct number is found.

- Bugs Found

During testing, I discovered three main bugs:

1. The score calculation logic in the update_score function was incorrect. The game was calculating the winning score incorrectly because of an off-by-one error. In addition, wrong guesses were not penalized consistently, causing some incorrect guesses to receive points instead of losing points.
2. The game allowed guesses outside the valid range of 1 to 100. It continued to provide "higher" and "lower" hints even when the player entered values below 1 or above 100, resulting in confusing gameplay.
3. The New Game command did not work correctly. Selecting New Game did not reset the game, and I had to refresh the browser page manually to start a new round.

- Fixes Applied

I fixed two bugs in the update_score function. First, I corrected an off-by-one error in the win score calculation by removing an unnecessary +1 from the formula. Second, I fixed the scoring logic so that both "Too High" and "Too Low" guesses apply the same penalty instead of incorrectly rewarding points for certain wrong guesses.

I also added range validation to prevent guesses below 1 or above 100 and verified that the hint system remained consistent. Finally, I fixed the New Game functionality so the game resets correctly without requiring the browser page to be refreshed.
